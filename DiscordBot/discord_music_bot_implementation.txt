## Day 1: Project Setup & Django Configuration (COMPLETE)

### Objectives
- Initialize Django project structure
- Set up virtual environment and dependencies
- Create Discord bot skeleton
- Configure database connection

### Tasks
1. **Install pipenv and create virtual environment**
   - `pip install pipenv`
   - `pipenv install Django djangorestframework discord.py yt-dlp PyNaCl requests django-mssql-backend pyodbc python-decouple`
   - `pipenv shell`

2. **Verify Django project structure**
   - Check `manage.py` exists in root
   - Verify `DiscordBot/` settings folder exists
   - Check existing `db.sqlite3` file

3. **Update Django settings.py** (`DiscordBot/settings.py`)
   - Add `'rest_framework'` to `INSTALLED_APPS`
   - Add `'queue_api'` to `INSTALLED_APPS`
   - Configure REST_FRAMEWORK settings:
   ```python
   REST_FRAMEWORK = {
       'DEFAULT_RENDERER_CLASSES': [
           'rest_framework.renderers.JSONRenderer',
       ],
       'DEFAULT_PAGINATION_CLASS': 'rest_framework.pagination.PageNumberPagination',
       'PAGE_SIZE': 50
   }
   ```

4. **Configure MSSQL database** (Optional - for MSSQL setup)
   - Create MSSQL database: `DiscordMusicBot`
   - Update `DATABASES` in settings.py:
   ```python
   DATABASES = {
       'default': {
           'ENGINE': 'mssql',
           'NAME': 'DiscordMusicBot',
           'USER': 'sa',
           'PASSWORD': 'your_password',
           'HOST': 'localhost',
           'PORT': '1433',
           'OPTIONS': {
               'driver': 'ODBC Driver 17 for SQL Server',
               'isolation_level': 'READ_COMMITTED',
           },
       }
   }
   ```

5. **Create queue_api Django app**
   - `python manage.py startapp queue_api`

6. **Create bot directory structure**
   - Create `bot/` directory
   - Create `bot/__init__.py`
   - Create `bot/main.py` (entry point)
   - Create `bot/api_client.py` (HTTP client)
   - Create `bot/commands/` directory
   - Create `bot/commands/__init__.py`

7. **Create .env file**
   ```
   DISCORD_BOT_TOKEN=your_actual_token_here
   DJANGO_SECRET_KEY=your_secret_key
   MSSQL_SERVER=localhost
   MSSQL_DATABASE=DiscordMusicBot
   MSSQL_USER=sa
   MSSQL_PASSWORD=your_password
   ```

8. **Create bot main file** (`bot/main.py`)
   - Set up Discord intents
   - Create bot instance with command prefix `/`
   - Add `on_ready()` event handler
   - Add cog loading system
   - Create `main()` async function

9. **Create test bot cog** (`bot/bot.py`)
   - Create `MusicBot` class inheriting from `commands.Cog`
   - Add `ping` command to test connectivity
   - Add `hello` command

10. **Run migrations**
    - `python manage.py migrate`

11. **Test Django server**
    - `python manage.py runserver`
    - Verify server runs on `http://localhost:8000`

### Deliverables
- Ôťů Django project configured and running
- Ôťů `queue_api` app created and registered
- Ôťů MSSQL connection configured (or SQLite ready)
- Ôťů Bot directory structure created
- Ôťů Bot skeleton with test commands ready

### Tasks
1. **Create GuildQueue model** (`queue_api/models.py`)
   - Create `GuildQueue` model with fields:
     - `guild_id` (CharField, unique, indexed)
     - `created_at` (DateTimeField, auto_now_add)
     - `updated_at` (DateTimeField, auto_now)
     - `is_playing` (BooleanField, default False)
     - `current_song_index` (IntegerField, default 0)
   - Add `__str__` method
   - Add `Meta` class with ordering

2. **Create Song model** (`queue_api/models.py`)
   - Create `Song` model with fields:
     - `queue` (ForeignKey to GuildQueue)
     - `title` (CharField)
     - `url` (URLField)
     - `added_by` (CharField)
     - `position` (IntegerField)
     - `duration` (IntegerField, nullable)
     - `added_at` (DateTimeField, auto_now_add)
   - Add `__str__` method
   - Add `Meta` class with ordering and unique_together constraint

3. **Create SongSerializer** (`queue_api/serializers.py`)
   - Create `ModelSerializer` for Song
   - Include fields: id, title, url, added_by, position, duration, added_at
   - Mark `id` and `added_at` as read_only

4. **Create GuildQueueSerializer** (`queue_api/serializers.py`)
   - Create `ModelSerializer` for GuildQueue

   - Include fields: id, guild_id, is_playing, current_song_index, songs, created_at, updated_at
   - Mark `id`, `created_at`, `updated_at` as read_only

5. **Register models in admin** (`queue_api/admin.py`)
   - Register `GuildQueue` model
   - Register `Song` model
   - Set list_display for each model

6. **Create and run migrations**
   - `python manage.py makemigrations queue_api`
   - `python manage.py migrate`

7. **Test models via Django shell**
   - `python manage.py shell`
   - Create test GuildQueue and Song instances
   - Verify serializers work correctly

### Deliverables

## Day 3: REST API Endpoints & ViewSets(CURRENT)

### Objectives
- Create ViewSets for Song and GuildQueue
- Configure DRF router for URL routing
- Build API endpoints for queue management

### Tasks
1. **Create GuildQueueViewSet** (`queue_api/views.py`)
   - Inherit from `viewsets.ModelViewSet`
   - Set `queryset = GuildQueue.objects.all()`
   - Set `serializer_class = GuildQueueSerializer`
   - Set `lookup_field = 'guild_id'`
   - Add custom action `add_song()` (POST):
     - Calculate position
     - Save song to queue
     - Return 201 with song data
   - Add custom action `current_song()` (GET):
     - Return currently playing song
     - Return 404 if none playing
   - Add custom action `next_song()` (POST):
     - Increment current_song_index
     - Save queue

2. **Create SongViewSet** (`queue_api/views.py`)
   - Inherit from `viewsets.ModelViewSet`
   - Set `queryset = Song.objects.all()`
   - Set `serializer_class = SongSerializer`
   - Override `destroy()` method:
     - Delete song
     - Reorder remaining songs
     - Return 204

3. **Configure routing** (`queue_api/urls.py`)
   - Import `DefaultRouter` from rest_framework
   - Create router instance
   - Register `GuildQueueViewSet` as 'queues'
   - Register `SongViewSet` as 'songs'
   - Include router URLs at `api/` prefix

4. **Update main URLs** (`DiscordBot/urls.py`)
   - Import include from django.urls
   - Add path for `api/v1/` including queue_api URLs

5. **Test all endpoints with Postman/curl**
   - GET `/api/v1/queues/` - List all queues
   - POST `/api/v1/queues/` - Create new queue
   - GET `/api/v1/queues/{guild_id}/` - Get specific queue
   - POST `/api/v1/queues/{guild_id}/add_song/` - Add song to queue
   - GET `/api/v1/queues/{guild_id}/current_song/` - Get current song
   - POST `/api/v1/queues/{guild_id}/next_song/` - Advance to next song
   - DELETE `/api/v1/songs/{id}/` - Delete song

### Deliverables
- Ôťů ViewSets created and working
- Ôťů All API endpoints functional
- Ôťů Custom actions responding correctly
- Ôťů HTTP status codes correct
- Ôťů Endpoints tested via Postman

### Time Estimate: 2-3 hours

## Notes

- **Environment**: Always activate pipenv shell before running commands: `pipenv shell`
- **Bot Token**: Never commit `.env` file to git. Add to `.gitignore`
- **Database**: Configure MSSQL connection in `settings.py` before running migrations
- **Discord Setup**: 
  1. Go to Discord Developer Portal
  2. Create Application
  3. Create Bot user
  4. Copy token to `.env`
  5. Enable "Message Content Intent" under Privileged Gateway Intents
  6. Invite bot to test server with OAuth2 URL generator
- **FFmpeg**: Ensure FFmpeg is installed and in PATH for audio playback
- **Testing**: Test API endpoints with Postman before connecting frontend
- **Development**: Run Django and bot in separate terminals
  - Terminal 1: `python manage.py runserver`
  - Terminal 2: `python bot/main.py`
- **Static Files**: For production, run `python manage.py collectstatic`
- **Debugging**: Enable Django debug mode in settings.py during development only
- **Error Logs**: Check terminal output for both Django and bot for debugging
- **Migrations**: Always run `python manage.py migrate` after model changes
- **Admin Panel**: Access at `http://localhost:8000/admin/` after creating superuser
